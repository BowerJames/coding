- **Ingest**: Added the wiki's first `pattern` page — [Stream seam](/patterns/streaming.md) (`type: pattern`) — and its Python implementation [Streaming seams in Python](/python/streaming.md) (`type: python`), from the user's `asyncio` stream-seam snippet. The pattern is the durable, language-agnostic design: a provider **returns a stream it fills from a background task** by splitting read/write authority at a seam — a single-pass consumer (`AsyncIterator`) and a producer (`push`/`end` writer) linked by one shared queue (`_Core`), paired by a `create_stream()` factory that returns `(consumer, writer)` so the provider keeps the writer and hands back the read side only. Captures the structure (Core/Consumer/Writer/factory), when-to-use, the design invariants (single-pass, single-consumer, **sentinel-terminated** — `None` = EOS, safe because events are never `None`; **idempotent close** — `push`/`end` no-op after `end`; the terminal event is delivered before iteration stops), the sentinel-vs-enum discussion (Rust/TS use `Option`/`Result`/discriminated variants instead of a reserved value), and limitations (no dedicated error channel in the basic form, backpressure depends on the queue, no fan-out). The Python page carries the snippet verbatim plus the provider idiom (`create_stream()` + `asyncio.create_task` + `return consumer`, with `end()` in a `finally`), and typing notes (PEP 695 generics set a **Python 3.12+ floor**, subsuming `Self`'s 3.11+; `TEvent | None`; `__slots__` everywhere; `from __future__ import annotations`). **Decision: `python/type-safety.md` intentionally untouched** — the snippet uses typing but is not a source *about* type safety, and the `None`-sentinel is a property of the pattern, so a cross-link would be manufactured (typing notes live as content on the python page instead). Refreshed root [index](/index.md): new **Patterns** group + the python page under Python.

- **Ingest**: Added [Type safety in Python](/python/type-safety.md) (`type: python`) — directive "type safety is critical; preferred type checker is mypy in strict mode." Four-rule policy (strict mode; annotate every function; avoid `Any` by receiving unknown data as `object` and **narrowing** it; `# type: ignore` justified, preferably unneeded). Safety hierarchy `Any` < `cast` < runtime-narrow (`isinstance`/`TypeGuard`/`match`, verified) — `cast` is a justified last resort.
